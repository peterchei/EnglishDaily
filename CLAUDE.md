# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**EnglishDaily** ("Mastering the London Flow") is a self-contained daily English learning system for a Cantonese speaker. A cron job on OpenClaw (Linux) runs `check_and_prepare_daily.js` each morning — it generates the lesson, audio, updates the PWA dashboard, pushes to GitHub Pages, and sends a Telegram summary.

Live app: https://peterchei.github.io/EnglishDaily/

## Agents

When the user says "check and push", "review and commit", or "commit if good" — delegate to the `code-reviewer` subagent (`.claude/agents/code-reviewer.md`). It runs `npm test`, reviews the diff, and commits/pushes only if all checks pass.

## Common Commands

```bash
# Run all unit tests (28 tests, Node built-in runner, no extra deps)
npm test

# Run a single test file
node --test test/lesson-utils.test.js

# Regenerate index.html, README.md, sw.js, manifest.json from lesson files
python update_index.py

# Run the full daily pipeline (requires env vars — see below)
node check_and_prepare_daily.js
```

Required env vars for `check_and_prepare_daily.js`:
```
GEMINI_API_KEY      # Gemini 2.0 Flash lesson generation (also tried for Google TTS)
TELEGRAM_BOT_TOKEN  # Telegram Bot API
TELEGRAM_CHAT_ID    # Recipient chat ID
EDGE_TTS_SCRIPT     # Optional: path to tts-converter.js (has a sensible default)
```

## Architecture

```
check_and_prepare_daily.js  # Cron entry point — full pipeline
lib/lesson-utils.js         # Pure utility functions (TTS script builder, Telegram formatter)
test/                       # Unit tests for lib/lesson-utils.js
update_index.py             # Regenerates the static site from lesson markdown files
lessons/                    # Lesson files: vocabulary_YYYY-MM-DD[_suffix].md
  2026-02/                  # Subfolder (older lessons)
  Lessons/                  # Subfolder (older lessons)
media/                      # Pronunciation audio: YYYY-MM-DD_pronunciation.mp3
state.json                  # Idempotency tracker (prevents duplicate daily deliveries)
docs/                       # Design and implementation documentation
```

**`check_and_prepare_daily.js`** — The entire daily pipeline in one script:
1. Read `state.json` → exit early if today already sent
2. Generate lesson via Gemini API → `lessons/vocabulary_YYYY-MM-DD.md`
3. Generate audio: Google TTS first, then edge-tts fallback, then skip if both fail
4. Run `python update_index.py`
5. `git add / commit / push`
6. Send Telegram summary (MarkdownV2 format)
7. Write `state.json` marking today as sent

**`lib/lesson-utils.js`** — Pure functions, no side effects, fully unit-tested:
- `buildTTSScript(markdown)` — builds English-only narration (strips Cantonese)
- `parseTelegramSummary(markdown, today)` — builds MarkdownV2 Telegram message
- `escapeMarkdownV2(text)` — escapes all 18 Telegram MarkdownV2 reserved characters

**`update_index.py`** — Scans `lessons/` recursively, builds `index.html` with inline `LESSONS_DATA` JSON (powers offline swipe navigation), regenerates `README.md`, `sw.js`, `manifest.json`. Must be run after adding a new lesson. Template slots: `{WORDS_COUNT}`, `{STREAK}`, `{LESSONS_DATA_JSON}`, `{HISTORY_HTML}`.

**`index.html` / `sw.js` / `manifest.json`** — Fully generated by `update_index.py`. Do not edit manually.

## Lesson File Format

New lessons go in `lessons/vocabulary_YYYY-MM-DD.md`:

```markdown
# Daily English Lesson - YYYY-MM-DD

## 1. **Word** (Part of Speech)
**Pronunciation:** /IPA notation/
**Definition:** English definition.
**Cantonese Meaning:** 廣東話

**Example 1:**
English example sentence.
**Cantonese Example 1:**
廣東話例子

**Example 2:**
Second English example.
**Cantonese Example 2:**
Second Cantonese translation.
```

When asked to create a lesson manually:
1. Create `lessons/vocabulary_YYYY-MM-DD.md` using the format above
2. Run `python update_index.py`
3. Commit and push — GitHub Pages auto-deploys

## PWA Dashboard

The Today tab supports swipe navigation across the last 7 base lessons. `_extra` and `_mega` lessons appear in History only. Swipe and arrow navigation both call `navigateLesson(direction)` in the generated JS.

## Key Notes

- `GEMINI_API_KEY` is often restricted to Gemini only in Google Cloud Console — Google TTS will fail with `API_KEY_SERVICE_BLOCKED`, and the script falls back to edge-tts automatically.
- Telegram uses MarkdownV2 — all dynamic content (dates, word names, Cantonese) must go through `escapeMarkdownV2()`. Reserved characters include `-`, `.`, `!`, `(`, `)`, `_`, and others.
- All paths use `__dirname` so the script runs on both Windows (local dev) and Linux (OpenClaw cron).
- `sw.js` cache version is a Unix timestamp — it is bumped automatically on every `update_index.py` run.
