## Project Overview

**EnglishDaily** ("Mastering the London Flow") is a personal English learning system for a Cantonese speaker. It manages daily English vocabulary/idiom lessons with bilingual (English + Cantonese) content, pronunciation audio, and a PWA dashboard.

Live app: https://peterchei.github.io/EnglishDaily/

## Daily Automated Workflow

This repo is driven by an AI agent running in the OpenClaw environment. The daily flow is:

1. **AI agent creates** `content/YYYY-MM-DD.md` with the day's vocabulary lesson
2. **AI agent runs** `python update_index.py` — regenerates `index.html` and updates `README.md`
3. **AI agent pushes** to GitHub — GitHub Pages auto-deploys the updated dashboard
4. **AI agent runs** `node check_and_prepare_daily.js` — generates TTS audio and sends a lesson summary via OpenClaw messaging

**To regenerate index.html** (works locally on Windows or Linux in OpenClaw):
```bash
python update_index.py
```

> `check_and_prepare_daily.js` still has hardcoded Linux paths (TTS script path, OpenClaw CLI) and is only meant to run in the OpenClaw environment.

## Architecture

```
content/          # New lessons: YYYY-MM-DD.md
lessons/          # Archive: vocabulary_YYYY-MM-DD[_suffix].md
  2026-02/        # Subfolder by month (older)
  Lessons/        # Named subfolder (older)
media/            # Audio: YYYY-MM-DD_pronunciation.mp3
state.json        # Tracks which lessons have been sent
```

**index.html** — Vanilla JS single-page app with three tabs (Today, History, Missions). Manually edited or regenerated by `update_index.py`. Contains the full lesson content inline.

**update_index.py** — Reads all markdown lesson files from both `lessons/` and `content/`, generates `index.html` and updates `README.md`. Must be run after adding a new lesson to update the dashboard.

**sw.js** — Service Worker for PWA offline caching. Cache version is a Unix timestamp in the variable name at the top.

## Lesson File Format

New lessons go in `content/` as `YYYY-MM-DD.md`. Older lessons are in `lessons/` as `vocabulary_YYYY-MM-DD.md` (with optional suffixes `_extra`, `_mega`).

Lesson markdown structure:
```markdown
# Daily English Lesson - YYYY-MM-DD

## 1. **Word** (Part of Speech)
**Pronunciation:** /IPA notation/
**Definition:** English definition
**Cantonese Meaning:** 廣東話

**Example 1:**
English example sentence.
**Cantonese Example 1:**
廣東話例子

**Example 2:**
Second example.
**Cantonese Example 2:**
Second Cantonese translation.
```

## When Asked to Create a Lesson

1. Create `content/YYYY-MM-DD.md` following the format above
2. Run `python update_index.py` to regenerate `index.html` and `README.md`
3. Commit and push to GitHub — Pages auto-deploys

Audio (`media/YYYY-MM-DD_pronunciation.mp3`) is generated by `check_and_prepare_daily.js` via edge-tts in the OpenClaw environment.

## Swipe Navigation (index.html)

The Today tab supports swipe-left/swipe-right to navigate between the last 7 base lessons. This is powered by a `LESSONS_DATA` JS array that `update_index.py` embeds inline — no external fetch needed (works offline).

- `_extra` and `_mega` lessons are excluded from swipe navigation (they remain in the History tab only)
- `update_index.py` calls `parse_markdown_content()` for each of the last 7 base lessons and serialises them as JSON into the `{LESSONS_DATA_JSON}` template slot
- Navigation arrows (`‹` / `›`) and touch swipe both call `navigateLesson(direction)` in the generated JS
